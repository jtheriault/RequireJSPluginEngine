RequireJS as a Plugin Engine: "Echo" Demonstration

Description

This demonstrates configuring RequireJS with different path values in order to swap implementations.

In this demonstration, the user is asked for input, which is confirmed and repeated to the screen.

The two implementations depend on three (implied) interfaces:

Prompt
a class get text input in response to a question. It takes the question in the constructor, displays it via a display method and it "returns" the response as a parameter to callback.
Confirm
a class get a boolean response to a question. It takes the question in the constructor, displays it via a display method and it "returns" the response as a parameter to callback.
Message
a class get a display a message. It takes the message in the constructor and displays it via a display method.

In Obnoxious Mode, the default, the RequireJS path for output is set to output/obnoxious which is a folder containing the implementations of the three interfaces as wrappers to the browser window's native prompt, confirm and alert functions.

In Polite Mode, the RequireJS path for output is set to output/polite which is a folder containing a implementations for the interfaces which rely on HTML elements for I/O.
